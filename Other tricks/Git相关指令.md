*<font color="#ffff00">出现云端版本不匹配的解决方案</font>*

|**步骤**|**指令**|**程序员的理解**|
|---|---|---|
|**第一步：拉取**|`git pull origin main --rebase`|把云端多出的内容下载下来，并将你的修改排在它们之后。|
|**第二步：处理（可选）**|`git rebase --continue`|如果出现冲突（Conflict），手动修改文件后执行此命令。若无冲突则跳过。|
|**第三步：推送**|`git push`|此时本地与云端已对齐，可以正常发货。|

***<font color="#9bbb59">每天上传GitHub</font>***
|**步骤**|**命令**|**程序员的理解**|
|---|---|---|
|**第一步：装箱**|`git add .`|
|**第二步：备注**|`git commit -m "感悟"`|说明，如 `"2026-03-05 算法练习"`。|
|**第三步：发货**|`git push`|

GitHub仓库合作上传流程：
1.git checkout main (确保自己在主分支)
2.git pull origin main 先把最新修改拉到本地
3.git checkout -b "name" (先为新功能建个临时工作区)
4.git status 查看状态，看什么文件改了、什么还没被保存
5.git add /*path*/ （只点名添加改动的文件)
6.git commit -m "feat:"
7.git push origin "name(名字同3)"

紧急状态处理：
1.commit了但没push: git reset --soft HEAD~1
2.已经push到GitHub上:git revert HEAD
3.强制回到与服务器一样的状态：git reset --hard origin/main
