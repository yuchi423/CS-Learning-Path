
| **指令**                                           | **功能**                            |
| ------------------------------------------------ | --------------------------------- |
| **`git config --global user.name "YourName"`**   | 设置全局用户名，用于标识提交者身份。                |
| **`git config --global user.email "YourEmail"`** | 设置全局邮箱，确保 GitHub 贡献统计（小绿格）正确。     |
| **`git init`**                                   | 在当前文件夹初始化一个本地 Git 仓库（档案室）。        |
| **`touch .gitignore`**                           | 创建一个忽略名单文件，防止私人配置被传到云端。           |
| **`git remote add origin [URL]`**                | 将本地仓库与远程 GitHub 仓库建立关联。           |
| **`git status`**                                 | **查看当前状态**。检查哪些笔记被修改了，哪些还没准备上传。   |
| **`git add .`**                                  | 将当前目录下所有修改过的笔记放入“暂存区”（准备发货）。      |
| **`git commit -m "感悟"`**                         | 为暂存区的修改提交一个快照，并附带一条更新说明。          |
| **`git branch -M main`**                         | 强制将当前分支重命名为 `main`，以对齐 GitHub 标准。 |
| **`git push -u origin main`**                    | 第一次推送并建立本地与云端的永久关联。               |
| **`git pull --rebase`**                          | 拉取云端更新并重新排在本地修改之后，解决版本冲突。         |
| **`git push`**                                   | **日常推送**。在建立关联后，一键同步本地笔记到 GitHub。 |
| **`git push -f`**                                | **强制推送**。用本地内容覆盖云端，仅建议在初期同步出错时使用。 |
| **git config --list**                            | **查看当前git的全局状态**                  |
| **`git remote -v`**                              | 查询**关联的 GitHub 网址。**              |
| `git rebase --abort`                             | **强行退出冲突状态**                      |

*<font color="#ffff00">出现云端版本不匹配的解决方案</font>*

|**步骤**|**指令**|**程序员的理解**|
|---|---|---|
|**第一步：拉取**|`git pull origin main --rebase`|把云端多出的内容下载下来，并将你的修改排在它们之后。|
|**第二步：处理（可选）**|`git rebase --continue`|如果出现冲突（Conflict），手动修改文件后执行此命令。若无冲突则跳过。|
|**第三步：推送**|`git push`|此时本地与云端已对齐，可以正常发货。|
