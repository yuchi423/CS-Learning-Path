*<font color="#ffff00">出现云端版本不匹配的解决方案</font>*

|**步骤**|**指令**|**程序员的理解**|
|---|---|---|
|**第一步：拉取**|`git pull origin main --rebase`|把云端多出的内容下载下来，并将你的修改排在它们之后。|
|**第二步：处理（可选）**|`git rebase --continue`|如果出现冲突（Conflict），手动修改文件后执行此命令。若无冲突则跳过。|
|**第三步：推送**|`git push`|此时本地与云端已对齐，可以正常发货。|

|**步骤**|**命令**|**程序员的理解**|
|---|---|---|
|**第一步：装箱**|`git add .`|
|**第二步：备注**|`git commit -m "感悟"`|说明，如 `"2026-03-05 算法练习"`。|
|**第三步：发货**|`git push`|

GitHub仓库合作上传流程：
1.git checkout main (确保自己在主分支)
2.git pull origin main 先把最新修改拉到本地
3.git checkout -b "name" (先为新功能建个临时工作区)
4.git status 查看状态，看什么文件改了、什么还没被保存
5.git add /*path*/ （只点名添加改动的文件)
6.git commit -m "feat:"
7.git push origin "name(名字同3)"

紧急状态处理：
1.commit了但没push: git reset --soft HEAD~1
2.已经push到GitHub上:git revert HEAD
3.强制回到与服务器一样的状态：git reset --hard origin/main

恢复到上一个存档点：git checkout .
查看所有历史版本：git log --oneline
git reset -- hard 版本号(eg:d4e309d)


# 本地 Git 版本管理指南

  

## 📋 核心命令速查表

  

| 功能 | 命令 | 说明 |

|------|------|------|

| **查看历史记录** | `git log --oneline` | 显示所有版本的简洁列表 |

| **查看完整历史** | `git log` | 显示完整的历史信息（包括时间、作者等） |

| **添加改动** | `git add .` | 把所有改动的文件加入待提交区 |

| **创建存档点** | `git commit -m "描述"` | 保存一个版本，必须有描述信息 |

| **查看当前状态** | `git status` | 查看有哪些文件被改动了 |

| **回到某个版本** | `git reset --hard <版本号>` | 彻底回到指定版本（改动会丢失）|

| **恢复单个文件** | `git checkout -- <文件路径>` | 只恢复某一个文件到上次提交状态 |

| **恢复全部文件** | `git checkout .` | 放弃所有改动，回到上个存档点 |

| **查看改动内容** | `git diff` | 显示具体改了哪些行 |

| **查看版本详情** | `git show <版本号>` | 查看某个版本做了什么改动 |

  

---

  

## 🔄 常见工作流

  

### 场景1：改完代码后，保存一个新的备份点

  

```bash

# 第1步：查看改了什么

git status

  

# 第2步：确认要改，就添加到待提交区

git add .

  

# 第3步：创建备份点

git commit -m "这里写一句简短的说明，比如：补充了主页的关于我部分"

```

  

### 场景2：代码改坏了，想回到上个备份点

  

```bash

# 方式1：恢复所有文件

git checkout .

  

# 方式2：只恢复某个文件

git checkout -- MyIntro.html

```

  

### 场景3：想看看之前的版本改了什么

  

```bash

# 第1步：查看所有版本

git log --oneline

  

# 第2步：选中某个版本号（比如 d4e309d），看它的改动

git show d4e309d

  

# 或者，想回到那个版本看看整个项目什么样

git reset --hard d4e309d

  

# 反悔了，想回到最新版本

git reset --hard master

```

  

---

  

## 💾 版本号说明

  

- 版本号是一串 7 位的字母数字，比如 `d4e309d`

- 每次 commit 时都会自动生成

- 用 `git log --oneline` 查看所有版本号

- 与任何版本号一起使用就能精准定位

  

---

  

## ⚠️ 重要提示

  

- 🔴 **`git reset --hard`** 会永久删除改动，要谨慎

- 🟡 **`git checkout .`** 只影响当前项目，改不了已提交的历史

- 🟢 **`git commit`** 前一定要写清楚描述，便于以后查找
